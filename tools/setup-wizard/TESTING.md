# TESTING.md — セットアップウィザード検証手順（Windows / 向井の実機）

井上さん（初回ユーザー・Windows想定）に渡す前に、向井の実機で以下を通しで確認する。
所要 30〜45 分。**検証には「普段使いでない GitHub テストアカウント」があると理想**
（Fork やガジェット作成が本アカウントに残らないため）。無ければ本アカウントでよい
（作られた Fork / gadgets/ フォルダは後で消せる）。

## 0. 事前準備（1回だけ）

- [ ] README.md の手順で **OAuth App を作成**し、`config.json` の `githubClientId` に記入
      （「Enable Device Flow」のチェックを忘れずに）
- [ ] リポジトリが **Public** になっていること（Private のままだと手順4の clone が失敗する）
- [ ] ルートで `npm install` 済みであること

## 1. 開発モードで通し確認

```powershell
npm start -w setup-wizard
```

| # | 操作 | 期待結果 |
|---|---|---|
| 1 | 起動 | 「ようこそ」画面。タイトルに表示名（config.json の appName）と所要時間 約15分 |
| 2 | 次へ → 環境診断 | git / Node.js の2項目が ✓（両方入っている場合） |
| 3 | 「再診断」 | 一覧が再取得される（結果は同じでOK） |
| 4 | 次へ → GitHub連携 → 「GitHubと連携する」 | ブラウザで github.com/login/device が開き、アプリに8桁コードが表示される |
| 5 | ブラウザにコードを入力して承認 | アプリ側が自動で「✓ 連携できました: <ユーザー名> さん」になり「次へ」が有効化 |
| 6 | 次へ → 「Fork → clone → npm install を実行」 | 黒いログ枠に進捗が流れ、数分後「✓ 準備完了: <パス>」（デフォルトはホーム直下 `nagaya-base`） |
| 7 | 次へ → ガジェットIDに `Test` と入力 | 「✗ 半角英小文字…」のリアルタイム警告、「作成」ボタンは無効のまま |
| 8 | IDを `test-room` に修正、表示名「テストの部屋」 | 「✓ 使えるIDです」→ 作成 → 「✓ gadgets/test-room を作成しました」 |
| 9 | 次へ → 「開発サーバを起動してブラウザで開く」 | ブラウザでダッシュボードが開き、「テストの部屋」ガジェットの承認画面が出る |
| 10 | 「AIツールに貼る最初の指示文」のコピー | クリップボードに指示文（gadget-spec を読んで〜）が入る |

### 失敗系の確認（重要）

- [ ] **手順4を認証せず放置** → アプリがエラーにならず「認証を待っています…」が続くこと
- [ ] **手順6を同じフォルダで2回実行** → 「フォルダが既に存在します」エラーと、
      「clone 済みフォルダを指定」で続行できること
- [ ] **手順8で既存の `_template` と同名や既存IDを入力** → 事前に✗になること
- [ ] 各エラー表示に「ここまでの状態」と「手動で続行する場合」が出ること
- [ ] （可能なら）Node をアンインストールした環境 or PATH を外した状態で環境診断 →
      ✗ 表示・「ダウンロードページを開く」で公式サイトが開くこと

## 2. インストーラのビルドと確認


- [ ] 別フォルダ（できれば別PC）で実行 → SmartScreen 警告 → 「詳細情報」→「実行」で
      インストールできる（この手順が README 記載どおりであること）
- [ ] インストール版でも手順1の通しが動く

## 既知の制限

- トークンはメモリ保持のみ。アプリを閉じると再認証が必要（仕様）
- ウィザードを閉じると開発サーバも止まる（画面6に注記あり。以後はターミナルで
  `npm run dev:gadget <ID>`）
- Mac 版のビルドは Mac 実機が必要（未検証。井上さんは Windows のため後回し）
