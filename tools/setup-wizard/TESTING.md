# TESTING.md — セットアップウィザード検証手順（Windows / 向井の実機）

**検証対象は配布物そのもの**（ポータブル exe をダブルクリックして起動）。
井上さん（初回ユーザー・Windows想定）が体験するのと同じ形で確認する。所要 30〜45 分。

検証には「普段使いでない GitHub テストアカウント」があると理想（Fork や
ガジェット作成が本アカウントに残らないため）。無ければ本アカウントでよい
（作られた Fork / clone / gadgets/ フォルダは後で消せる）。

## 0. 最初に必ず: 未署名警告の突破手順（配布ページにもこのまま載せる）

exe をダブルクリックすると、初回は青い画面
**「Windows によって PC が保護されました」**（SmartScreen）が出ます。これは
コード署名をしていないためで、次の手順で起動できます:

1. 青い画面の **「詳細情報」** をクリック（左寄りの小さいリンク）
2. 表示された **「実行」** ボタンをクリック

※「実行」が出ない場合は、exe を右クリック → プロパティ → 下部の
「許可する」にチェック → OK でも解除できます。
※ ダウンロード直後はブラウザ側（Edge/Chrome）も警告することがあります。
「保持する」→「詳細表示」→「保持する」で保存してください。

### 所有者パス・冪等性の確認

- [ ] **所有者（yosuke175）でログインして手順6** → ログに「Fork は不要なので本家を
      直接使います」と出て、Fork されずに clone される

### 失敗系の確認（重要）

- [ ] **手順4を認証せず放置** → アプリがエラーにならず「認証を待っています…」が続くこと
- [ ] **同名フォルダが「リポジトリでない」場合**（空フォルダを nagaya-base という名で
      作っておく）→ 「リポジトリではないようです」エラーで別の保存先へ誘導されること
- [ ] **手順8で既存IDを入力** → 事前に✗になること
- [ ] 各エラー表示に「ここまでの状態」と「手動で続行する場合」が出ること
- [ ] （可能なら）Node が入っていない環境 or PATH を外した状態で環境診断 →
      ✗ 表示・「ダウンロードページを開く」で公式サイトが開くこと

## 補足

- 開発中の動作確認は exe を作らず `npm start -w setup-wizard` でも可（ただし
  最終確認は必ず配布物で行うこと）
- ポータブル形式は起動時に一時フォルダへ自己展開するため、**初回起動は数秒〜十数秒
  かかる**（2回目以降も同程度。インストーラー形式より起動は遅い）
- トークンはメモリ保持のみ。アプリを閉じると再認証が必要（仕様）
- ウィザードを閉じると開発サーバも止まる（画面6に注記あり。以後はターミナルで
  `npm run dev:gadget <ID>`）
- Mac 版のビルドは Mac 実機が必要（未検証。井上さんは Windows のため後回し）
