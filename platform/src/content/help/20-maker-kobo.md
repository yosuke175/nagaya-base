# 工房の使い方

**要約**: 工房（あなたの作業場）は、道具（ガジェット）を作る・直す・公開する作業をまとめた画面です。工房そのものはWebの画面なので、ここでファイルを作るのではなく、**AIへの「指示文（貼り付ける文章）」を出す**のが役割です。指示文をクラウドの作業場のAIに貼って作り込み、できたら「部屋で試運転」して確認し、よければ公開申請します。

はじめての方は、先に案内所の「はじめての道具づくり（手順）」で全体の流れをつかんでください。この記事は、その工房画面の中身を一つずつ説明します。

## 工房はどこにある？

長屋にログインし、メニューの「工房」を開きます。ログインしている環境でのみ使えます（軒先＝ゲストでは使えません）。

工房の画面は、上から順に次のブロックが並んでいます。

- 道具をつくる（入口）
- あなたの道具（過去に作った道具の管理）
- 新しい道具をつくる（お題→AI指示文）
- 部屋で試運転（プレビュー）
- AI設定
- 道具市の道具を「自分の道具」として登録

以下、順に説明します。

## 道具をつくる（入口）

いちばん上のブロックです。「はじめての道具づくり（手順書）を開く」ボタンから、GitHubのアカウント作成から始まる手順書（案内所の「はじめての道具づくり」）に飛べます。はじめての方はまずここです。

その下に「かんたんな流れ」が4ステップで書かれています。要点は次のとおりです。

- GitHub にアカウントを作り、本家を**自分のGitHubにコピー（Fork）**する
- そのコピーを**クラウドの作業場**（Codespaces / Claude Code 等）で開いてAIと作る
- GitHub に push（保存）してから、工房の指示文でAIと作り込む
- 「部屋で試運転」で確認し、よければ PR を出して公開申請する

「自分のPC（ローカル）で作りたい場合はこちら」という折りたたみもあります。開くと**セットアップウィザード**のダウンロードが出ます。これは任意の選択肢で、PCに開発環境を自動でそろえたい人向けです。クラウドで作るなら不要です。

## あなたの道具（過去作の管理）

あなたが「自分の道具」として登録した道具が一覧で並びます。各道具には、名前・道具のID・状態が表示されます。状態は次の4つです。

| 表示 | 意味 |
|---|---|
| 構築中 | まだ道具市に出していない（作りかけ・非公開） |
| 審査中 | 審査の途中（該当時のみ） |
| 公開中 | 道具市に並んでいる |
| 停止中 | 大家により停止された |

### 公開／非公開の切り替え

各道具の右側にボタンがあります。

- 状態が「公開中」のとき: **非公開にする**ボタン（道具市から下ろす）
- それ以外のとき: **道具市に公開する**ボタン（道具市に並べる）

公開の切り替えは、道具の持ち主（owner）本人か大家だけができます。公開申請そのものの正式な流れ（PR→マージ）は、案内所の「道具市に公開する（PR）」を参照してください。

### 改善する／複製して新規

各道具のカードの下に、2つのボタンがあります。押すと**AIへの指示文**が下に出ます（この画面でコードを書くのではなく、指示文をコピーして使います）。

- **改善する**: いまの道具に手を入れる指示文が出ます。指示文の中の「（ここに直したいこと・足したい機能を書く）」を、自分の言葉に置き換えて使います。
- **複製して新規**: いまの道具を土台にして別の新しい道具を作る指示文が出ます。先に「新しい道具のID」（半角英小文字・数字・ハイフン）を入れると、その名前で複製する指示文になります。

指示文の右下の「指示文をコピー」を押すと文章がコピーされます。これを、クラウドの作業場で動くAI（Claude Code など）に貼り付けて進めます。指示文の使い方は案内所の「クラウドで道具を作る」を参照してください。

道具のカード画像や表示文（道具市での見た目）は、この工房ではなく「道具市」の各カードの「表示を編集」から変更します。

## 新しい道具をつくる（お題→AI指示文）

ゼロから新しい道具を作るブロックです。

### 手順

- お題のボタンを選びます（「おみくじ」「今日の3つ」「プロンプト帳」「単位変換」、または自分のアイデアで作る「白紙」）
- お題を選ぶと、下に「道具のID」の入力欄が出ます（半角英小文字・数字・ハイフン、3〜40文字）
- IDを入れると、雛形（`_template`）を複製して作り込むための**AI指示文**が下に表示されます
- 「指示文をコピー」でコピーし、クラウドの作業場のAIに貼って作ります

指示文には、`docs/gadget-spec.md`（道具の説明書）と `gadgets/_template/`（雛形）を読ませる指示や、雛形を複製するコマンド、守るべき制約（SDK経由でのみ通信する等）が最初から入っています。中身を書き換える必要はほとんどありません。

## 部屋で試運転（プレビュー）

GitHubに push した作りかけを、公開せずに自分の部屋の中で本物として動かして確認するブロックです。GitHubユーザー名・ブランチ・道具のIDを入れて試運転します。詳しくは案内所の「部屋で試運転（プレビュー）」を参照してください。

## AI設定

道具の中でAIを使う（文章生成など）には、あなた自身のAI APIキーの登録が必要です。ここから登録・確認できます。仕組みとキーの取り方は案内所の「AIの使い方」を参照してください。

## 道具市の道具を「自分の道具」として登録

道具市には並んでいるのに、まだ「あなたの道具」一覧に入っていない道具があると、このブロックが出ます。あなたが作った道具を選んで「自分の道具として登録」を押すと、その道具の持ち主（owner）があなたになり、公開状態や表示を管理できるようになります。
