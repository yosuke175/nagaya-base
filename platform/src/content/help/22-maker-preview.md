# 部屋で試運転（プレビュー）

**要約**: 作りかけの道具（ガジェット）を、**公開する前に**、自分の部屋の中で本物として動かして確かめられます。これが「部屋で試運転（プレビュー）」です。GitHubの自分のコピー（Fork）に push した内容を、GitHubユーザー名・ブランチ・道具のIDの3つを入れて呼び出します。試運転しても道具市には並びません（まだ公開ではありません）。

先に案内所の「クラウドで道具を作る」で、道具を作って push するところまで進めておいてください。

## 試運転とは（何が起きるのか）

試運転では、あなたのForkに push 済みの `gadgets/<道具のID>/` を、長屋が自分のオリジン（本番と同じ場所）で読み込み、**本番とまったく同じ環境**で動かします。保存（storage）・AI・外部サービス連携も本番同様に動くので、公開前でも「実際どう動くか」を正確に確かめられます。

大事な点が2つあります。

- **push した内容だけが反映されます。** クラウドの作業場で編集しただけ（push前）では反映されません。
- **試運転は公開ではありません。** 道具市に並ぶのは、PR → マージ → 公開 の後です。試運転は自分だけに見えます。

## 試運転のやり方

工房を開き、「部屋で試運転（プレビュー）」のブロックに、次の3つを入れます。

| 入力欄 | 何を入れる | 例 |
|---|---|---|
| GitHubユーザー名 | 自分のGitHubのユーザー名（Forkの持ち主） | inoue-taro |
| ブランチ | 作業しているブランチ名（ふつうは main） | main |
| 道具のID | `gadgets/` の下のフォルダ名（manifest の id と同じ） | my-first-gadget |

3つを正しく入れると「この道具を部屋で試運転する」ボタンが押せるようになります。押すと、画面いっぱいの窓が開き、その中であなたの道具が本物として動きます。窓の上には「試運転中: 道具のID」「ユーザー名@ブランチ・未公開」と表示されます。確認が終わったら「閉じる」で戻ります。

GitHubユーザー名とブランチは次回のために覚えられます（同じ端末なら次から入力済みになります）。

### 入力のときの注意

- 道具のIDは、**半角英小文字・数字・ハイフンのみ、3〜40文字**です（フォルダ名・manifest の id と一致させます）
- ブランチ名を変えて作業している場合は、その名前を正しく入れてください（ふつうは `main`）

## push した内容が反映される

編集して確認、の流れはこうです。

- クラウドの作業場（または PC）で編集する
- **GitHubの自分のリポジトリに push（保存）する**
- 工房の「部屋で試運転」でボタンを押す（または窓を開き直す）

push を忘れると、直したはずの内容が試運転に出てきません。「直したのに変わらない」と思ったら、まず push できているか確認してください。

## うまく動かないときの直し方

試運転で思ったとおりに動かないときは、あわてず次のように直します。

- **どこがどうおかしいか**を具体的にメモする（例: ボタンを押しても何も出ない／保存した内容が消える）
- クラウドの作業場のAIに「ここが動かない」と、その症状を伝える
- AIが直したら、**もう一度 push する**
- 工房で**もう一度試運転**して確認する

これを、納得できるまで「直す → push → 試運転」で繰り返します。

読み込み自体に失敗したときは、窓の中に「ガジェット『道具のID』を読み込めませんでした」というメッセージが出ます。その場合は、道具のID（フォルダ名）・ユーザー名・ブランチのどれかが違っていないか、そもそも push できているかを見直してください。

## 試運転できたら公開へ

試運転で納得できたら、いよいよ道具市への公開申請です。流れ（PR・CLA・マージ・公開ボタン）は案内所の「道具市に公開する（PR）」を参照してください。
