# 権利のはやわかり

正文は [CONTRIBUTING.md](https://github.com/yosuke175/nagaya-base/blob/main/CONTRIBUTING.md) と
[CLA.md](https://github.com/yosuke175/nagaya-base/blob/main/CLA.md)。ここは早見表です。

## まず読む早見表（行為ベース）

- 長屋の道具を使う・学ぶ — **○ 自由**（商売でも社内利用は自由）
- 自分が書いたコードを転用する — **○ 自由**（あなたの著作物）
- 他人のコードを読んで学び、ゼロから書く — **○ 自由**（アイデアは著作権の外）
- 他人のコードをコピーして製品の主要部分に — **△ 稟議へ**（原作者の同意＋管理者の許諾）
- プラットフォームごと複製して販売 — **✕ 不可**（協会の商用権）

AI に書き直させてもコピーはコピーです（著作権は「表現」を保護し、
書き直しの手数では消えません）。売る物には他人のコードを混ぜず、
学んで自分で書くのが基本です。

## 権利の要点（CLA・ライセンス）

- あなたのコードの著作権は**あなたに帰属したまま**
- 自分の貢献は本プロジェクトの外でも自由に使える
- プロジェクト側は商用を含む利用許諾を持つ（BSL 1.1 / 4年後に Apache 2.0）
- 収益の分配はなし。クレジット表記で貢献を記録
- 第三者の権利を侵害しないことはあなたの責任（**AI 生成コードも同様**）

## 商用利用の相談（稟議）

他人のコードを有償の製品に組み込みたいときは、事前に GitHub Issue
「商用利用の相談」を立ててください（①原作者の同意 → ②管理者の許諾）。
自分のコードのみを使う場合、稟議は不要です。
