# 道具（ガジェット）の作り方

道具づくりは、**AI にまかせるのがいちばん簡単**です。ここはAIの勉強会でもあります。
ぜひAIを相棒にしてください。プログラミングの経験は要りません。

## いちばん簡単な方法：AI にまかせる（おすすめ）

Claude（Claude Code）、Codex、Antigravity などの「AIコーディングツール」を使うと、
会話するだけで道具が形になります。**PCへのインストールは不要**で、クラウドで進められます。

1. GitHub に**自分のコピー（Fork）**を作る（自分のGitHubに長屋の複製ができます）
2. そのコピーを**クラウドの作業場**（GitHub Codespaces や Claude Code 等）で開く
3. AIに `docs/gadget-spec.md`（道具の作り方の説明書）と雛形（`gadgets/_template`）を読ませ、「こういう道具が作りたい」と伝える
4. できたら GitHub に push → 長屋の**「工房 → 部屋で試運転」**で、作りかけを自分の部屋で動かして確認する
5. よければ**提案（Pull Request）**を出して公開申請する

あとはAIが「これはこう作れます」とどんどん提案・実装してくれます。
**工房の「新しい道具をつくる」に、AIに最初に貼る指示文**が用意してあるので、それをコピーして使うと迷いません。
うまくいかないところも、AIに「ここが動かない」と伝えれば直してくれます。
（自分のPCで作りたい方は、メニュー「案内」→「職人のはじめ方」からセットアップウィザードも使えます。）

## 道具からできること（安全な範囲で）

道具は**安全な枠の中（サンドボックス）**で動きます。これは「他の道具やあなたのデータに
勝手に触れない、隔離された小部屋」のこと。宣言していない機能や通信先は使えないので、
悪意のない不具合が混ざっても長屋全体には広がりません。

その枠の中で、次のことができます（使う機能は道具ごとに宣言します）:

- **保存領域** — その道具×あなた専用の保存場所（他の人・他の道具からは見えない）
- **AI** — 文章を作らせる（使う人が登録したAIキーを消費。Claude / OpenAI / Google など）
- **音声入力** — マイクから文字にする
- **外部サービス連携** — 使う人が自分のキーを登録して外部サービスにつなぐ

## できた道具を長屋に出す（公開）

1. まず**「工房 → 部屋で試運転」**で、作りかけを自分の部屋で動かして確認します（まだ公開はされません）
2. 納得したら「提案」として送ります（GitHub の Pull Request ＝「これを追加したいです」という申請）
3. 取り込まれ（マージされ）、工房で「道具市に公開する」を押すと、道具市に並び、みんながインストールできます

※ 将来は提案時に自動チェック（マニフェスト検証・宣言外API検査・ビルド）を挟む予定です（現在は準備中）。

※ この「提案」や自動チェックは職人（作る人）向けの仕組みです。
使うだけの店子は気にしなくて大丈夫です。

## AI を使わずに手作業で作る場合

AIを使わなくても作れます（その分、少し手間がかかります）:

1. 雛形フォルダをコピーする
2. `manifest.json` に道具の名前・使う機能を書く
3. HTML と JavaScript で中身を作る
4. `npm run dev:gadget 自分のID` で動かして確認する
5. 提案（Pull Request）を送る
