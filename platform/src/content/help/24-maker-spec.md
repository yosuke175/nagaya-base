# 道具の仕様（やさしい版）

**要約**: 道具（ガジェット）は、長屋の画面の中の「安全な枠」の中で動く小さなWebアプリです。守るべき約束は主に4つ。**manifest.json**（道具の名札＝どんな道具で何を使うかの宣言）、**gadget-sdk**（長屋と話すための決まった窓口）、**サンドボックス**（安全な枠）、**BYOK**（外部サービスは使う人自身のキーで）です。この記事はやさしい要約です。正確な仕様は `docs/gadget-spec.md`（道具の説明書）を参照してください。

道具づくりでは、この説明書をAIに読ませるのが早道です（案内所の「クラウドで道具を作る」参照）。読者は仕組みを丸暗記する必要はありません。「そういう約束がある」と分かれば十分です。

## 道具ってどういうもの？

- 長屋の画面の中の枠（`iframe`）の中で動く、独立した小さなWebアプリです
- HTML + CSS + JavaScript で動くものなら何でもOK（AIに全部書かせてもOK）
- 長屋の機能（データ保存など）は、決まった窓口（**gadget-sdk**）を通してだけ使えます

## manifest.json（道具の名札）

各道具のフォルダに必ず1つ置く「名札」です。長屋との約束を書いた紙で、ここに書いたことしか道具はできません。おもな項目は次のとおりです。

| 項目 | 何を書く |
|---|---|
| id | 道具のID。フォルダ名と同じにする（半角英小文字・数字・ハイフン、3〜40文字） |
| name / description | 道具市に出す名前と説明（日本語でOK） |
| version | バージョン番号。公開申請のたびに上げる |
| permissions | 使う機能の宣言（下の「権限」参照）。宣言していない機能は使えない |
| size | 部屋の棚に置いたときの大きさ（small / medium / large / full） |
| externalServices | 外部サービスを使うときの宣言（下の「外部連携」参照） |

ほかに author（作者）、entry（最初に開くHTML）、icon（道具市カードの画像・任意）があります。正確な書き方は `docs/gadget-spec.md` の「マニフェスト」の章にあります。

### 権限（permissions）

道具が使える長屋の機能は、manifest で先に宣言します。宣言していない機能を呼ぶと、その場で拒否されます。おもな権限は次のとおりです。

| 権限 | できること |
|---|---|
| storage | この道具専用の保存領域を使う |
| notify | 通知を表示する |
| profile | 使う人の表示名を取得する（IDやメールは渡らない） |
| ai | AIによる文章生成を使う（使う人が登録したAPIキーを消費） |
| microphone | 音声入力のためにマイクを使う |
| ai-tools | 案内AIから呼べる「道具の操作」を提供する（任意・宣言しなければ無関係） |

店子（使う人）は、道具を棚に並べる前に、この権限の一覧を見て承認します。**あとから権限を増やすと、既にインストール済みの人には再承認が求められます。**

## gadget-sdk（長屋と話す窓口）

道具は、長屋のデータベースを直接いじることはできません。代わりに、配布される **gadget-sdk** という決まった窓口を通して話します（裏側では postMessage という仕組みで、枠の中と外が安全にメッセージをやり取りしています）。

SDKでできることの例です。

- `storage` — この道具×この人の分だけ保存・読み出し（合計1MBまで）
- `notify` — 通知を出す
- `ui.resize` / `ui.getTheme` — 大きさを変える・明暗テーマを知る
- `user.getProfile` — 使う人の表示名を得る（IDやメールは得られない）
- `ai.complete` — AIに文章を作らせる（キーは長屋が預かり、道具には渡らない）
- `services.getCredential` — 外部サービス用のキーを取り出す（BYOK。下記）

呼び出しは全部「少し待つ」タイプ（非同期）です。使い方のコード例は `docs/gadget-spec.md` の「SDK の使い方」にあります。

## サンドボックス（安全な枠）

道具は「サンドボックス」＝**安全な枠**の中で動きます。これは、道具が悪さをできないようにする仕切りです。枠の中からは、次のことは**できません**。

- 長屋のデータベースを直接いじる
- ほかの道具のデータを見る
- 使う人のログイン情報（トークン）を取り出す
- manifest で宣言していない場所（ドメイン）への通信
- 外側の画面を勝手に操作する・ポップアップを出す

この仕切りのおかげで、店子は知らない作者の道具でも安心して試せます。仕切りは作る人が意識して守るものではなく、長屋が自動でかけています。宣言した約束の範囲で作れば、自然とこの枠に収まります。

## 外部連携は BYOK（使う人自身のキーで）

道具が外部のサービス（例: AIのAPIなど）とつながるときは、manifest の `externalServices` で**通信先をあらかじめ宣言**します。宣言していない場所への通信は自動でブロックされます。

外部サービスの認証は **BYOK（Bring Your Own Key＝使う人が自分のキーを用意する）** 方式です。キーは店子が「連携設定」で自分で入力し、長屋が暗号化して預かります。**キーは道具のコードには渡りません。** 使う人が迷わないよう、宣言には貼り付け形式のヒント（`setupHint`）か設定手順へのリンク（`setupUrl`）を付けます。

なお、Google ドライブ全体のような広すぎるアクセスは、方針として要求できません。詳しくは `docs/gadget-spec.md` と、外部連携の方針を定めた設計文書を参照してください。
