# 道具市に公開する（PR）

**要約**: 試運転で納得できた道具（ガジェット）を、みんなが使えるように道具市へ出す手順です。GitHubで「PR（プルリクエスト）＝本家に入れてくださいという申請」を出し、初回は CLA（作者の同意書）に署名し、本家に取り込まれ（マージされ）たら、工房で「道具市に公開する」を押します。試運転（案内所「部屋で試運転」）だけでは公開されません。公開は必ずこの申請を通ります。

## 全体の流れ

- GitHubで **PR（プルリクエスト）** を作る＝本家への公開申請
- 初回は **CLA（同意書）** に署名する
- 本家に取り込まれる（**マージ**される）
- 工房で **「道具市に公開する」** を押す
- 道具市に並び、みんながインストールできるようになる

## PR（プルリクエスト）とは

PR（プルリクエスト）は、「自分のコピー（Fork）で作った変更を、本家に取り込んでください」とお願いする申請です。あなたのForkの `gadgets/あなたの道具ID/` を本家に入れる、というお願いになります。

### 手順

- GitHubで、自分のFork（`あなたのユーザー名/nagaya-base`）を開く
- 「Pull request」を作る（本家 `nagaya-base` に向けて）
- 何の道具か、簡単な説明を書いて送信する

作業場のAIツール（Claude Code など）にPR作成を頼めることもあります。うまくいかないときは、管理人・ベテラン職人に「PRの出し方が分からない」と聞いてください。

## CLA（同意書）への署名

**初めてPRを出すときは、CLA（Contributor License Agreement＝作者の同意書）への署名が必要です。** これは、あなたが作ったコードを NAGAYA-BASE に載せてよい、という取り決めです。PRを出すと案内が出るので、それに従って一度署名すれば、次回以降は不要です。

CLAは公開の入り口を守る大事な仕組みです。試運転（プレビュー）はこの署名を経ずに動く自分専用の確認機能で、**公開の裏口にはなりません**。公開したいときは必ずPRとCLAを通ります。

## マージ（本家に取り込まれる）

PRが本家に取り込まれることを「マージ」といいます。マージされると、その道具のコードが本家に入ります。

補足として、公開の前に自動でチェックする仕組み（マニフェストの検証、宣言していないAPIを呼んでいないかの検査、ビルドが通るかの確認）は**準備中（予定）**です。整うと、PRのたびに自動でチェックが走るようになります。それまでは、作った内容が仕様（案内所「道具の仕様（やさしい版）」）に沿っているか、自分と周りで確認してください。

## 工房で「道具市に公開する」を押す

マージされただけでは、まだ道具市には並びません。最後に、あなた自身が公開スイッチを入れます。

### 手順

- 長屋にログインし、メニュー「工房」を開く
- 「あなたの道具」の一覧で、その道具を見つける（一覧にない場合は、下の「道具市の道具を『自分の道具』として登録」から登録する）
- **「道具市に公開する」** ボタンを押す

状態が「公開中」になれば完了です。道具市に並び、店子（使う人）がインストールできるようになります。公開を取り下げたいときは、同じ場所の「非公開にする」で下ろせます。工房画面の詳しい説明は案内所の「工房の使い方」を参照してください。

## 更新するとき

一度公開した道具を直したいときも、流れは同じです。手を入れて push し、`manifest.json` の `version`（バージョン番号）を上げて、またPRを出します。権限や外部の通信先を増やした場合は、既にインストール済みの店子に**再承認**が求められます（安全のための仕組みです）。バージョンや権限の考え方は案内所の「道具の仕様（やさしい版）」を参照してください。

## もし公開後に問題が見つかったら

道具市は「作った人を信じて、問題があれば後から対処する」運びです。公開後に問題が見つかった場合、大家（管理者）が道具を即時停止できます。停止したときは理由が作者に伝えられます。
