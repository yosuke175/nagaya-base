# debug-console — デバッグコンソール

大家（管理者）向けの点検用ガジェット。プラットフォーム連携を1つずつ実際に叩いて、
結果とエラーをそのままログに出す。不具合の切り分け用。

## 何が分かるか

- **① AI疎通テスト**: `gadget.ai.complete` を単体で呼ぶ。ここが成功するのに個々の
  ガジェット（スケジュール秘書など）で失敗するなら原因はそのガジェット側、ここで
  失敗するならAI設定キー／プラットフォーム側が原因、と切り分けられる。送るメッセージは
  `gadget.storage` に記憶される。
- **② ストレージ疎通**: `gadget.storage` の set→get 往復。
- **③ 環境情報**: URL・userAgent・オンライン状態・Web Speech 対応可否など。
- **ログ**: すべての操作を時刻つきで記録。「全部コピー」でそのまま貼り付けできる。
- iframe 内の未捕捉エラー（`window.onerror` / `unhandledrejection`）も拾ってログに出す。

## 案内AIからも呼べる（ADR-011）

- `run_diagnostics`（read）: 環境情報をまとめて返す。
- `echo`（read）: 渡した文字列をそのまま返す。案内AI→ガジェットのツール実行経路の疎通確認用。

## 開発

```bash
npm run dev:gadget debug-console
```

## メモ

- カタログ（道具市）は `gadgets/*/manifest.json` から自動生成される（`_`/`.` 始まりの
  ディレクトリは除外）。このディレクトリを置くだけで道具市に並ぶ。
- 外部サービスのクレデンシャルはガジェットごとに分離されているため、この道具から
  他の道具（スケジュール秘書等）のクレデンシャルは読めない（設計どおり）。
